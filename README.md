# Artemis Cosmos Unofficial Patch
Mod for the Artemis Cosmos Spaceship Bridge Simulator game that provides only bugfixes, with no frills.

###  To download the latest version:
1) Go to https://github.com/quaris628/ArtemisCosmosUnofficialPatch/releases/latest
2) Download the "Source Code" zip file.

### To install:
1) Unzip the mod files (right-click the zip file you downloaded, then select "Extract All...")
2) Browse to your Artemis Cosmos folder that contains the game files that you wish to apply this mod to. If you bought Artemis Cosmos through steam, you can open this folder from your library by right-clicking Artemis Cosmos in the list of games (on the left), hover over "Manage", then select "Browse local files".
3) Optional, but strongly recommended: Create a backup copy of your entire Artemis Cosmos folder.
4) Take the mod files you extracted and copy them into your Artemis Cosmos folder, such that the mod files overwrite the vanilla files.
5) A window should appear that says the destination already has files with the same names. (If this doesn't happen, then something went wrong in step 4.) When this happens, choose to replace the files in the destination.

### To report bugs, give feedback, etc:

If you believe the bug/feedback/etc is only pertinent to this mod, then create a public issue on github ( https://github.com/quaris628/ArtemisCosmosUnofficialPatch/issues ) or you may privately email me at `quaris314@gmail.com`.

If you believe the bug/feedback/etc is only pertinent to vanilla Artemis Cosmos, then create a github issue here: https://github.com/artemis-sbs/LegendaryMissions/issues/new/choose

If you are unsure whether the bug/feedback/etc is pertinent to this mod or vanilla, then assume it's pertinent to this mod; I should be able to sort out which is which, and if it's vanilla I'll forward it.

# List of bugs patched:

General/Misc
- Fix crashes and some other bugs triggered by random corruptions in graphics files when they're auto-generated by the game, by providing uncorrupted copies of all of these files (vanilla issues [#422](https://github.com/artemis-sbs/LegendaryMissions/issues/422), [#606](https://github.com/artemis-sbs/LegendaryMissions/issues/606), and [#392](https://github.com/artemis-sbs/LegendaryMissions/issues/392))
- Fix crashes and many other very serious bugs that are triggered by typing unusual characters into text input boxes, by disallowing typing unusual characters (vanilla issue [#569](https://github.com/artemis-sbs/LegendaryMissions/issues/569))
  - For some text fields, this fix introduces other visual bugs when typing invalid characters. However, since these bugs are much tamer (alibet more obvious), I think this a clearly good tradeoff.
- Fix the ` character being present in text input boxes that should be empty (vanilla issue [#641](https://github.com/artemis-sbs/LegendaryMissions/issues/641))
- Make GUIBoxDataSafeBackup.txt be the same as the original GUIBoxData.txt
- Make legendary missions available in remote_mssion_pick

Engineering
- Remove unrespected duplicate keybind entries for engineering presets from hot_key_settings.txt (vanilla issue [#401](https://github.com/artemis-sbs/LegendaryMissions/issues/401))
- Resolve keybind conflict by adding grid_control_grid_button_9 keybind entry to hot_key_settings.txt, bound to Ctrl+9 (vanilla issue [#642](https://github.com/artemis-sbs/LegendaryMissions/issues/642))
- Add grid_control_grid_button_0 keybind entry to hot_key_settings.txt, bound to Ctrl+0
- Friendly bases and deepstrike resupply ship will no longer resupply energy in excess of 1000
- Prevent possible crashes when rotating and refocusing beams with engineering grid comms buttons

Science
- Remove unrespected duplicate keybind entries for science next and previous scan types from hot_key_settings.txt (vanilla issue [#621](https://github.com/artemis-sbs/LegendaryMissions/issues/621))

Communications
- Add keybinds for navigating comms sorted list to hot_key_settings.txt

Single-seat craft (Shuttles, Fighters, Bombers)
- Reasonably position zomm control widget on the single-seat craft cockpit screen (vanilla issue [#547](https://github.com/artemis-sbs/LegendaryMissions/issues/547))
- Resolve keybind conflict by changing default keybind for cycling single-seat craft's shield balance from O to K (vanilla issue [#328](https://github.com/artemis-sbs/LegendaryMissions/issues/328))
- Do not increase player single-seat craft shield strength fourfold (vanilla issue [#458](https://github.com/artemis-sbs/LegendaryMissions/issues/458))
- Do not add fighters to player ships that are configured to only have shuttles (vanilla issue [#368](https://github.com/artemis-sbs/LegendaryMissions/issues/368))
- Give Command, Industrial and Science bases some fighter and bomber bays, so that their refit times are more reasonable (vanilla issue [#517](https://github.com/artemis-sbs/LegendaryMissions/issues/517))
- Prevent possible crashes when selecting a single-seat craft's home dock location
- Minor performance improvement when spawning single-seat craft (which generally happens at the start of the game)
- Remove hangar_crafts.yaml's broken shields field (vanilla issue [#535](https://github.com/artemis-sbs/LegendaryMissions/issues/535))
- Remove hangar_crafts.yaml's required torp_types field, and instead always use ordinance type maximums from shipData.yaml (vanilla issue [#534](https://github.com/artemis-sbs/LegendaryMissions/issues/534))
- Remove hangar_crafts.yaml's usually-duplicative display_text field, and instead calculate craft type display text from the craft's roles

Gamemaster
- Allow opening the gamemaster console while the simulation is paused (vanilla issue [#638](https://github.com/artemis-sbs/LegendaryMissions/issues/638))
- Correct inaccuracies in the description of the game master sandbox map [#576](https://github.com/artemis-sbs/LegendaryMissions/issues/576))
